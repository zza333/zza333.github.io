[NOI Online 2021 提高组] 岛屿探险 题解

2021-03-27 5 min read # 信

题意简述

  • 一行 nnn 对数，第 iii 对数为权值对 <ai,bi><a_i,b_i><ai​,bi​>。
  • 回答 qqq 个询问 <lj,rj,cj,dj><l_j,r_j,c_j,d_j><lj​,rj​,cj​,dj​>，询问的是在 ljl_jlj​ 到 rjr_jrj​ 内所有数对中满足 aixor⁡cj≤min⁡(bi,dj)a_i\operatorname{xor} c_j\leq\min(b_i,d_j)ai​xorcj​≤min(bi​,dj​) 的数对个数。
  • 1≤n,q≤1051\leq n,q\leq 10^51≤n,q≤105，1≤lj≤rj≤n1\leq l_j\leq r_j\leq n1≤lj​≤rj​≤n，0≤ai,bi,cj0\leq a_i, b_i, c_j0≤ai​,bi​,cj​, dj≤224−1d_j\leq 2^{24} - 1dj​≤224−1。

解法

​ 先考虑一个询问怎么求。

​ 看到异或就知道大概是要建 0-1 trie 的。对于一个询问，分 bi≥djb_i\geq d_jbi​≥dj​ 和 bi<djb_i<d_jbi​<dj​ 两种情况考虑。

​ 对于 bi≥djb_i\geq d_jbi​≥dj​ 的情况，要求即为 aixor⁡cj≤dja_i\operatorname{xor} c_j\leq d_jai​xorcj​≤dj​ ，以 aia_iai​ 为权值建出序列的 0-1 trie。

​ 令 cjc_jcj​ 从高位往低位走，对于一个节点（假设 cjc_jcj​ 当前位为 000）有 000 和 111 两个儿子：若 djd_jdj​ 当前位为 111，则 000 儿子及其子树一定可行，111 儿子可能可行，向 111 递归；反之 111 儿子一定不可行，向 000 递归。这一部分是 0-1 trie 经典问题，不再赘述。

​ 对于 bi<djb_i< d_jbi​<dj​ 的情况，要求即为 aixor⁡cj≤bia_i\operatorname{xor} c_j\leq b_iai​xorcj​≤bi​ 。

​ 分析一下各种情况，这里还是假设 cjc_jcj​ 当前位为 000：

  • aia_iai​ 当前位为 000，bib_ibi​ 当前位为 000：可能可行，需向下递归。
  • aia_iai​ 当前位为 000，bib_ibi​ 当前位为 111：一定可行，子树全部可行。
  • aia_iai​ 当前位为 111，bib_ibi​ 当前位为 000：一定不可行，子树全部不可行。
  • aia_iai​ 当前位为 111，bib_ibi​ 当前位为 111：可能可行，需向下递归。

​ 观察发现：当 aixor⁡bi=0a_i\operatorname{xor}b_i=0ai​xorbi​=0 时需要向下递归，否则要么全可行，要么全不可行。

​ 对 cjc_jcj​ 当前位为 111 的情况分析类似。

​ 以 aixor⁡bia_i\operatorname{xor}b_iai​xorbi​ 为权值建出序列的 0-1 trie，再在每个节点记录其子树内 ai=0a_i=0ai​=0 和 111 的数的个数。查询时令 cjc_jcj​ 从高位往低位走，按上面的讨论做即可。

​ 再考虑原问题。按 bi,djb_i,d_jbi​,dj​ 排序，顺着扫一遍，分块维护两组 trie 树：一组是关于 bi≥djb_i\geq d_jbi​≥dj​ 的，另一组是关于 bi<djb_i< d_jbi​<dj​ 的，扫的时候在 trie 上动态加点删点即可。

​ 设块长为 BBB，值域为 AAA，时间复杂度 O(nlog⁡A+Q(nBlog⁡A+B))O(n\log A+Q(\frac{n}{B}\log A+B))O(nlogA+Q(Bn​logA+B))。n,Qn,Qn,Q 同级，理论取 B=nlog⁡nB=\sqrt{n\log n}B=nlogn​，实践中设 B=2000∼3000B=2000\sim 3000B=2000∼3000 时最优。空间 O(nlog⁡A)O(n\log A)O(nlogA)。

​ 较小的常数使得分块做法的通过成为可能。

​ CDQ 分治之类的做法（似乎）也可以类似地套用。按 bi,djb_i,d_jbi​,dj​ 排序，分治时左区间对右区间的贡献即所有 bib_ibi​ 的插入（bi<djb_i< d_jbi​<dj​ 部分的 trie 树上）和删除（ bi≥djb_i\geq d_jbi​≥dj​ 的 trie 树上）。其它题解里有这方面的详细讲解。

​ 不济的话把分块（时间被卡）和树套树（空间被卡）结合一下把块长调大一点也能过。

​ 放一下关键位置的源代码（bi<djb_i< d_jbi​<dj​ 部分的 trie 树建立与查询）：

void trieadd1(int u,int aa,int bb,int np){
	if(!np)return;
	int ng=(((aa^bb)&(1<<(np-1)))>>(np-1));
	if(!son[u][ng])son[u][ng]=++pb;
	dq[son[u][ng]][(aa&(1<<np-1))>>(np-1)]++;
	trieadd1(son[u][ng],aa,bb,np-1);
}
int triereq1(int u,int x,int np){
	if(!u)return 0;
	if(!np)return dq[u][0]+dq[u][1];
	int ng=((x&(1<<(np-1)))>>(np-1));
	return triereq1(son[u][ng],x,np-1)+dq[son[u][ng^1]][ng];
}
