初步分析

​ nemo 肯定走直线。

​ 不妨把 nemo 想象成只能以速度 vvv 运动或静止。而在最优策略中，nemo 显然不会静止不动。

​ 所以 nemo 的最优策略一定取如下形态：以某种顺序把食物排成一列 {pi}\{p_i\}{pi​}，从 {p1}\{p_1\}{p1​} 开始一个一个吃，每次都走当前最短时间对应路径。

​ 因为 nemo 已知了各食物的位置和速度，nemo 可以在目标食物的路径上某点等待，而最优的该点必然满足 nemo 和食物同时到达。因而可以计算出其到单个食物所需的最短时间。

​ 简单的高中解三角形问题，这里不再赘述。设 nemo 与食物距离为 sss，食物运动的方向与 nemo 相对于食物的方向夹角为 θ\thetaθ，nemo 速度为 vvv，食物速度为 uuu，联合运用向量点积和余弦定理得出 nemo 走到食物的最短时间为

t=sucos⁡θ±u2(cos⁡2θ−1)+v2u2−v2t=s\frac{u\cos\theta\pm\sqrt{u^2(\cos^2\theta-1)+v^2}}{u^2-v^2} t=su2−v2ucosθ±u2(cos2θ−1)+v2​​

​ 等式成立性与加减号取法：

  • 当 Δ=u2(cos⁡2θ−1)+v2<0\Delta=u^2(\cos^2\theta-1)+v^2<0Δ=u2(cos2θ−1)+v2<0 或 cos⁡θ<0&&u>v\cos\theta<0\&\& u>vcosθ<0&&u>v 时，nemo 追不上。
  • 否则取减号一定更优并且 ttt 必为正。证明就是用 Δ≥0\Delta\ge 0Δ≥0 化一下。
  • 特别地，u=vu=vu=v 特判掉。

​ 这道题唯一折磨人的点。

​ 考虑一个贪心策略：每次在当前所有能吃的食物中按某种关系选出最优的那一个并吃掉。重复该流程直至找不到可以吃的食物为止。

​ 设对于一个食物 iii，其权值为 wiw_iwi​，nemo 吃掉它所需时间为 tit_iti​，nemo 吃掉它时 nemo 的位置距当前位置为 did_idi​。

​ 肯定会有一些感性的认识：wiw_iwi​ 越大越好，tit_iti​ 越小越好，did_idi​ 越小越好，witi\frac{w_i}{t_i}ti​wi​​ 越大越好 ⋯\cdots⋯

​ 各人有各人的偏好，各种调参方法都能过，但调参的过程无一例外的折磨。这里题解采用

ki=witi2+1K1di+K2k_i=\frac{w_i}{t_i^2}+\frac{1}{K_1d_i}+K_2 ki​=ti2​wi​​+K1​di​1​+K2​

​ 每次选 kik_iki​ 最大的吃。K1,K2K_1,K_2K1​,K2​ 为两个 [1,10][1,10][1,10] 的随机数。

​ 多次贪心取最优解。实测在 10 次之内可以出答案，所花时间不超过 5s。

double eat2(){
	while(!pq.empty())pq.pop();
	double ct,nans=w0,nt=0;nx=fx,ny=fy,pa=0;
	for(int i=1;i<=n;i++){
		fl[i]=0;
		if(w[i]<nans){
			ct=calnt(i,nt);
			pq.push(make_pair(w[i]/(ct*ct)+G/dis(nx,ny,ax[i]+nt*mx[i],ay[i]+nt*my[i])+rand()%10,i));
		}
	}
	while(!pq.empty()){
		int ng=pq.top().second;pq.pop();
		fl[ng]=1,ct=calnt(ng,nt);
		if(nt+ct>T)break;
		nx=ax[ng]+(nt+ct)*mx[ng];
		ny=ay[ng]+(nt+ct)*my[ng];
		na[++pa]=(eve){nt+ct,nx,ny,ng};
		nans+=w[ng];
		nt+=ct;
		while(!pq.empty())pq.pop();
		for(int i=1;i<=n;i++)if(w[i]<nans&&!fl[i])ct=calnt(i,nt),pq.push(make_pair(w[i]/(ct*ct)+G/dis(nx,ny,ax[i]+nt*mx[i],ay[i]+nt*my[i])+rand()%10,i));
	}
	if(nans>aans){
		aans=nans,pans=pa;
		for(int i=1;i<=pa;i++)ans[i]=na[i];
	}
	return nans;
}
